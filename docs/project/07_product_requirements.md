# 07. Product Requirements (MVP)
**문서 버전:** 1.1  
**업데이트:** 2025-12-07  
**문서 목적:**  
AI Career Harvest OS의 MVP 및 확장 기능에 대한 제품 요구사항을 정의하여  
개발·디자인·데이터 팀이 동일한 방향성 아래 제품을 설계·구현하고,  
내부 AI 프롬프트 테스트/실험 도구까지 일관되게 구축할 수 있도록 한다.

---

## 1. Product Vision

AI 기반의 ‘성과 정제 및 커리어 자산화 시스템’을 제공하여,  
사용자가 단순히 “일을 했다” 수준의 기록을 넘어서 구조화된 성과(PACRI)로 자동 변환되도록 돕는다.

- 사용자는 **그냥 한 번에 주저리 적는다.**  
- 시스템은 이를 **Event 분해 → 성과 후보/Seed → PACRI 성과 → 프로젝트별 축적**까지 자동으로 처리한다.  

장기적으로는 사용자의 커리어 성장 패턴을 분석하여  
포트폴리오, 면접 답변, 연봉 협상 자료까지 자동 생성하는  
**‘AI 커리어 오퍼레이팅 시스템’**으로 확장한다.

---

## 2. Target User

### Primary Persona (IT 지식 노동자)

- Frontend / Backend / Mobile / DevOps / Data Engineer  
- Product Designer / UX Designer  
- PM / PO / Data Analyst  

특징:
- 프로젝트 단위로 일하고, “뭘 했는지”는 기억나는데  
  **“이걸 성과 문장으로 어떻게 써야 할지”** 막막한 사람  
- 이력서/포트폴리오/면접 준비 시즌마다 벼락치기하는 사람

### Secondary Persona

- 마케터, CS, 운영, 기획 등 비 IT 지식 노동자  
- 정량적 성과를 잡기 어려운 직무

---

## 3. Core Value Proposition

1. ✍ **일기처럼 써도 자동으로 성과가 분해된다**  
   - 긴 텍스트 입력 → Event 단위로 나눠서 성과 후보/Seed 자동 판별  

2. 🔎 **AI가 성과(Outcome)와 씨앗(Seed)을 구분해준다**  
   - 지금 당장은 결과가 없어도, “나중에 연결될 수 있는 문제/인사이트”를 Seed로 보관  

3. 🧠 **최소 질문만으로 ‘쓸만한 성과 문장(PACRI)’까지 안내**  
   - “질문 공포”가 아니라, “딱 1~2개만 물어보고 결과 뽑아주는” UX  

4. 📚 **프로젝트 기반 커리어 타임라인**  
   - 프로젝트별로 어떤 문제를 발견했고, 나중에 어떻게 해결했는지를 한 눈에  

5. 🧾 **보고서/포트폴리오/면접 답변에 바로 쓸 수 있는 출력물**  
   - PACRI → STAR 형식 변환, PDF/Notion용 텍스트 제공

## 4. Key Features (MVP)

### 4.1 성과 입력 (Raw Input)

- 한 번의 텍스트 박스에 **자유롭게 주저리 입력**  
- 하루/한 주 단위로 “이렇게저렇게 했다”를 쓰는 수준

요구사항:
- 모바일/데스크톱 모두에서 편하게 입력 가능
- 5~20줄 정도의 텍스트를 부담 없이 쓸 수 있는 UI

---

### 4.2 이벤트 자동 분해 & 1차 분류 (Phase 1)

- 기능:
  - 텍스트를 Event 단위로 나눔  
  - 각 Event에 대해:
    - `achievement_ready`, `needs_context`, `not_achievement`  
    - `missing_info` (["impact", "result"] 등)  
    - `is_seed_candidate` 플래그

- UI:
  - Event 리스트를 카드 형태로 보여주고:
    - 필요 없는 건 삭제
    - 필요하면 병합
    - 프로젝트 할당(단일/복수 선택)

---

### 4.3 프로젝트 매핑

- 사용자는 먼저 **프로젝트(카테고리)** 를 정의:
  - 예: “신규 온보딩 개편”, “코어 웹 서비스”, “사이드 프로젝트 블로그”  
- Event별로 프로젝트 할당:
  - 최소 1개 이상
  - 동일 Event가 여러 프로젝트에 속할 수 있음

- UX 포인트:
  - “현재 진행 중인 프로젝트 목록”을 먼저 보여주고  
  - 이벤트 카드에 한 번에 태그처럼 붙이기

---

### 4.4 Seed / Outcome 엔진

- `needs_context` + “문제/이슈/발견” 성향의 Event → **Seed**  
- 이미 Problem/Result/Impact가 있는 Event → **Outcome 후보**

MVP 정책:
- Seed/Outcome의 자동 생성은 AI 엔진에 맡기되,  
- 사용자가 Seed/Outcome 여부를 수동으로 토글할 수 있도록 지원

---

### 4.5 Phase 2 – 질문 & 맥락 보완

- `needs_context` 이벤트에 대해:
  - 1개의 핵심 질문 생성 (impact/result 위주)
  - 사용자는:
    - 답변하기
    - 다른 질문 요청
    - 그만 질문하기 선택 가능

- 그만 질문하기:
  - 해당 Event는 **Draft Outcome**으로만 생성
  - `quality_score`가 낮게 책정되고, 추후 보완 안내

---

### 4.6 PACRI 구조화 (Phase 3)

- Event + Q&A를 기반으로 PACRI 생성:
  - Problem / Approach / Contribution / Result / Impact  
  - title, skill_tags, quality_score 포함

- UX:
  - 생성된 PACRI를 카드 형태로 보여주고
  - 사용자가 한두 문장 직접 수정 가능
  - “이력서에 추가하기” / “포트폴리오에 저장하기” 버튼 제공 (v2)

---

### 4.7 성과 리스트 & 프로젝트 뷰

- 기능:
  - 프로젝트별로 Outcome/Seed 목록을 볼 수 있는 리스트
  - 필터:
    - 프로젝트
    - 기간 (주/월/분기)
    - 상태 (Seed / Draft / Finalized)

- 핵심:
  - “이 프로젝트에서 내가 뭘 했는지”를 한 페이지에서 보여주는 것

---

### 4.8 리포트 생성 (간단 버전)

- MVP에서는:
  - 프로젝트/기간 기준으로 PACRI들을 모아서  
  - 마크다운 또는 텍스트 블록으로 요약 출력

- 예:
  - “2025년 Q1 – 온보딩 개선 프로젝트 성과 요약”  
  - 각 Outcome 제목 + 한 줄 Impact + 주요 수치

PDF, 포트폴리오 템플릿, 면접 답변 자동 생성은 v2 이상에서 확장.

---

## 5. 내부용 기능: AI 테스트 랩 (Prompt Playground)

> 이건 “실제 유저 기능”이 아니라,  
> 세운이 지금 하려는 것처럼 **프롬프트/정책 실험용 내부 페이지**.

### 5.1 목표

- 샘플 텍스트를 입력해서  
  - Phase1 → Phase2 → Phase3 전체 파이프라인을 한 번에 돌려보고  
  - 각 단계 결과(JSON)를 비교/저장할 수 있는 도구.

### 5.2 요구사항

- 여러 개의 AI 버전/프롬프트 세트를 동시에 비교:
  - v1 vs v1.1 vs 실험용 B안 등
- 각 Phase 결과 저장:
  - raw_input, events, answers, pacri, seeds, outcomes
- “이 케이스에서 뭘 잘했고/망했는지”를 눈으로 확인하고  
  - 프롬프트를 개선할 수 있게 하는 UI

> 이 기능은 `ai-spec.md`, `05_data_model.md`, `06_ai_engine.md`에 정의된  
> 형식을 그대로 사용한다.

## 6. Non-Functional Requirements

### 6.1 성능

- 1회 입력(최대 100줄) 기준:
  - Phase1(Event 분해 + 분류): < 3초  
  - Phase2(질문 생성): 이벤트당 < 1초 (비동기/지연 허용)  
  - Phase3(PACRI 생성): 이벤트당 < 2초 (스트리밍 허용)

### 6.2 비용

- 기본은 `gpt-4o-mini` 우선 사용  
- PACRI 고품질 재작성은 “사용자 요청시”에만 `gpt-4o` 호출  
- 월간 AI 비용 목표:
  - MVP: 월 20~50달러 이하
  - 유저 수 증가 시에도 이벤트 수 기준으로 선형 상승

### 6.3 보안

- 텍스트 기반 MVP에서:
  - 민감 정보(전화번호, 계정 ID 등)는 클라이언트에서 1차 마스킹 권장
- 향후 Vision 도입 시:
  - 이미지 업로드 전, 민감 정보 블러 기능 탐색

---

## 7. Success Metrics (MVP)

- 1회 입력 → **Outcome 1개 이상 생성 비율 80%+**  
- PACRI 품질(내부 자체 평가) 평균 **75점 이상**  
- 베타 유저 1달 사용 후:
  - “이 서비스 덕분에 성과 정리가 쉬워졌다” 응답 70% 이상  
- 30일 재방문율 40% 이상

---

## 8. Future Expansion (요약)

- AI 면접 시뮬레이터 (PACRI → STAR 질문 자동 생성)  
- 역량/스킬 성장 그래프  
- Seed 기반 자동 OKR/목표 추천  
- 팀 계정 + 리뷰 모드 (리더가 팀원의 성과 리뷰)
